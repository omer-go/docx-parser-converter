DOCX PARSER CONVERTER - DOCUMENT_MODELS.PY DOCUMENTATION
========================================================

FILE PURPOSE:
This file defines Pydantic data models for representing the overall document structure in DOCX files. It provides models for document margins and the complete document schema that contains all document elements including paragraphs and tables.

IMPORTS:
- pydantic.BaseModel, pydantic.Field: For data validation and modeling
- typing.List, typing.Optional, typing.Union: For type hints
- docx_parser_converter.docx_parsers.models.paragraph_models.Paragraph: For paragraph elements
- docx_parser_converter.docx_parsers.models.table_models.Table: For table elements

FUNCTIONS:
None defined in this file.

CLASSES/DATA MODELS:

1. DocMargins(BaseModel)
   Purpose: Represents the margins of a document section
   Fields:
   - top_pt (Optional[float]): The top margin in points
   - right_pt (Optional[float]): The right margin in points
   - bottom_pt (Optional[float]): The bottom margin in points
   - left_pt (Optional[float]): The left margin in points
   - header_pt (Optional[float]): The header margin in points
   - footer_pt (Optional[float]): The footer margin in points
   - gutter_pt (Optional[float]): The gutter margin in points
   Description: Models the page margins for a document section, including standard page margins as well as header, footer, and gutter margins. All measurements are in points.

2. DocumentSchema(BaseModel)
   Purpose: Represents the overall structure of a document, including paragraphs and tables
   Fields:
   - elements (List[Union[Paragraph, Table]]): The list of document elements (paragraphs and tables) (required)
   - doc_margins (Optional[DocMargins]): The margins of the document
   Description: Top-level model that represents the complete document structure. Contains an ordered list of document elements (paragraphs and tables) and optional document margin settings.

DEPENDENCIES:
- Pydantic for data validation and modeling
- paragraph_models module for Paragraph data model
- table_models module for Table data model
- Standard typing module for type hints

DATA RELATIONSHIPS:
- DocumentSchema contains a list of Union[Paragraph, Table] elements
- DocumentSchema optionally contains DocMargins
- This represents the hierarchical structure where a document contains multiple elements of different types

USAGE CONTEXT:
These models represent the top-level structure of a DOCX document after parsing. The DocumentSchema serves as the root container for all document content, while DocMargins provides page layout information. This is typically the final output structure after parsing a complete DOCX document.
